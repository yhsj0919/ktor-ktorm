# 基础开发框架开发计划

目标：参考 RuoYi 等成熟后台框架，先完成一个可复用、可扩展、可验证的 Ktor + Ktorm 基础开发框架，再继续添加具体业务功能。

## 管理规则

- `[x]`：代码、静态检查和必要测试均已完成。
- `[ ]`：尚未完成。
- 基础能力优先注册为 Ktor 插件。
- 纯转换和无状态能力优先使用扩展函数或顶层函数。
- 业务 `service` 暂不作为框架收尾任务修改。
- 配置中心、多租户和复杂业务功能暂不进入主线。

## 当前状态

- 技术栈：Kotlin 2.4、Ktor 3.5、Ktorm 4.2、MySQL、HikariCP、Redis、Flyway。
- 包结构已经完成分层整理。
- 当前主要阻塞：无。全局 Gradle 缓存目录仍存在权限问题，已使用项目内缓存完成验证。
